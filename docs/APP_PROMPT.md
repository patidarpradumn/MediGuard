# Prompt for Building AI Healthcare Assistant App

Create a complete AI Healthcare Assistant web application.

The app should not give medical diagnosis. It should only suggest doctor specialization, risk level, emergency warning, first-care advice, and nearby hospital option based on user symptoms.

## Required Features

1. User can enter symptoms in a text box.
2. App analyzes symptoms using rule-based AI logic.
3. App suggests doctor specialization:
   - Fever, cough, cold, headache → General Physician
   - Skin rash, itching, allergy → Dermatologist
   - Tooth pain, gum pain → Dentist
   - Eye pain, blurry vision → Ophthalmologist
   - Chest pain, breathing issue → Emergency Care / Cardiologist
   - Stomach pain, vomiting, acidity → Gastroenterologist
   - Stress, anxiety, depression → Psychologist / Psychiatrist
4. App shows risk level:
   - Normal
   - Urgent
   - Emergency
5. Emergency symptoms should show red warning:
   “This may be an emergency. Please visit the nearest hospital immediately.”
6. App should show basic first-care advice.
7. App should include a nearby hospital search button using Google Maps search link.
8. App should include a clear disclaimer:
   “This app does not provide medical diagnosis. Please consult a qualified doctor.”
9. UI should be clean, modern, responsive, and healthcare-themed.
10. Code should have proper folder structure and beginner-friendly comments.

## Tech Stack

Frontend: HTML, CSS, JavaScript
Backend: Python Flask
AI Logic: Rule-based symptom matching
Future Upgrade: NLP/ML model and Google Maps API

AI Healthcare Assistant
